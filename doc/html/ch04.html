There are two ways to customize the presentation of your CV or résumé: stylesheet parameters and localized messages.

Stylesheet Parameters

XSL stylesheets can change their behavior based on user-specified parameters. For example, a parameter such as roman.headings=true might tell a stylesheet to use Roman numerals for chapter headings.

Specifying parameters in CV Sieve requires a bit more explanation because, as mentioned in the Introduction, it translates an HR-XML Resume file using a two-stage process. First, the CV Sieve stylesheets convert the CV or résumé into a DocBook file. Next, the DocBook stylesheets transform this file into a PDF, HTML, or plain text document. Thus, to fully control the output of CV Sieve, you need to specify two different sets of parameters: one for CV Sieve and another for DocBook.

Table 4.1, “Stylesheet Parameters” lists the parameters that can be used in CV Sieve.[3] The last column in the table indicates whether the parameter is for CV Sieve or DocBook.

Table 4.1. Stylesheet Parameters

ParameterDescriptionPossible ValuesType
month.formatFor date elements, this parameter indicates how the month should be displayedshort an abbreviated month, such as "Sep."; long the full month nameCV Sieve
generate.tocWhether to include a table of contents in the CV or résuméarticle toc generates a table of contents; article nop suppresses it.DocBook

The type of parameter (CV Sieve or DocBook) is important because it dictates how one specifies the parameter. For instance, if you are using the included helper script, a CV Sieve parameter would be specified in the cvsieve.params property, while a DocBook parameter would be specified in the docbook.params property. Refer to the build.properties files in the examples directory to see how this is done.


[3] Currently, there are only two parameters, but future versions of CV Sieve should offer more.
