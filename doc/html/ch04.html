Generating a CV or résumé with HR-XSL is a two-stage process: First, HR-XSL generates a DocBook version of the original file; next, this DocBook file is transformed into a PDF, HTML, or plain text version. Thus, to fully control the look of your CV or résumé, you need to customize the two stages individually, depending on the kind of customization you want. For content modifications (e.g., names of headings, date format, etc.), you would customize the HR-XSL transformation. For appearance modifications (e.g., font sizes, whether page numbers are displayed, etc.), you would customize the DocBook transformation.

HR-XSL Customization

There are two ways to customize the output of HR-XSL: stylesheet parameters and localized messages.

Stylesheet Parameters

XSL stylesheets can change their behavior based on user-specified parameters. For example, a parameter such as roman.headings=true might tell a stylesheet to use Roman numerals for chapter headings. Table 4.1, “HR-XSL Stylesheet Parameters” lists the parameters that can be used in HR-XSL.[3]

Table 4.1. HR-XSL Stylesheet Parameters

ParameterDescriptionPossible Values
month.formatFor date elements, this parameter indicates how the month should be displayedshort an abbreviated month, such as “Sep.”; long the full month name

The method of specifying these parameters depends on how you use the stylesheets. If you are using the included HR-XSL helper script, a parameter would simply be specified in the hr-xsl.params property. (Refer to the build.properties files in the examples directory to see how this is done.) For other methods, you will need to refer to the documentation of your stylesheet processor.

Localized Messages

All of the HR-XSL stylesheets have been localized. Simply stated, this means you can easily change the output of HR-XSL to a different language. To do so, go to the xsl/common/messages directory and make a copy of the en.xml file, giving it a name that represents your language. For example, if you want to create a French localization, you should use the two-letter language code for French, fr. You can then edit this new fr.xml file and translate all the English messages to their French equivalents.[4]

Once you have created a localized message file, you can tell HR-XSL to use it simply by setting the XSL parameter lang to the appropriate language code. Continuing with the French example above, if you are using the included Ant helper script, you would change the résumé example to French by adding the following string to the hr-xsl.params variable of the build.properties file:

lang=fr


[3] Currently, there is only one parameter, but future versions of HR-XSL should offer more.

[4] If you have created a localization file for your language, please consider submitting it to the HR-XSL maintainer so that it can be included in the distribution for others to use.
